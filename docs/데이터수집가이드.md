# 데이터 수집 가이드 (팀 배포용)

E&S 커뮤니케이션 시뮬레이터의 **페르소나 지식 베이스**를 여러 명이 나눠서 모을 때 쓰는 규칙·양식·자동화 방법입니다.

---

## 0. 큰 그림

```
수집처 등록          자동 수집             검수·승인            변환
data/sources.csv → harvest.py →  candidates.csv → (사람 검수) → knowledgeBase.json
  (RSS/URL 목록)    (템플릿 형식 초안)    verified=Y 로 승격      (앱이 사용)
```

- **담당은 페르소나별로** 나눕니다. (출처 유형이 아니라)
- 모든 자료는 **동일한 컬럼 포맷**으로 정리합니다.
- 자동 수집은 "초안"까지만. **사람 검수를 거친 것만** 앱에 들어갑니다.

---

## 1. 역할 분담

| 역할 | 인원 | 하는 일 |
|---|---|---|
| 페르소나 오너 | 페르소나당 1명 | 담당 페르소나 자료 수집·정리 (기후부/산업부/한전/가스공사) |
| 검수자 | 1명(교차) | 출처 실재·요약 충실성·티어·중복 확인 → `verified=Y` |
| 메인테이너 | 1명 | 수집처(sources) 관리, harvest 실행, JSON 변환·커밋 |

|---|---|---|---|

## 2. 출력 포맷 (컬럼 정의) — `data/collection-template.csv`

**한 행 = 한 자료.** 컬럼 순서·이름을 절대 바꾸지 마세요 (자동 변환이 이 순서에 의존).

| 컬럼 | 필수 | 규칙 |
|---|---|---|
| `persona` | ✅ | 기후부 / 산업부 / 한전 / 가스공사 |
| `source_type` | ✅ | 기고 / 논문 / 보고서 / 보도자료 / 언론기사 / 인터뷰 |
| `tier` | ✅ | 기고·논문·보고서=`t1`, 보도자료=`t2`, 언론기사·인터뷰=`t3` |
| `title` | ✅ | 원문 제목 그대로 |
| `org` | ✅ | 발행처(부처/언론사/저널) |
| `author` | | 필자·발언자 |
| `date` | ✅ | **`YYYY-MM-DD`** 고정 |
| `url` | ✅ | 접속 확인된 원문 링크 |
| `summary` | ✅ | ★모델이 읽는 유일한 본문★ — 핵심 입장·주장 **2~3문장, 숫자·날짜 포함** |
| `excerpt` | ✅ | 원문 **직접 인용 1~2문장** (토씨 그대로, 창작 금지) |
| `priority_axis` | ✅ | 이 자료가 뒷받침하는 페르소나 우선순위 |
| `stance_signal` | | 우호 / 우려 / 중립 |
| `collector` | ✅ | 수집자 이름 |
| `verified` | ✅ | 검수 완료 Y/N |
| `note` | | 비고(추정·한계 등) |

### 작성 규칙 (품질의 핵심)

1. **`summary`는 "제목 요약"이 아니라 "그 자료의 입장"** 을 쓴다.
   - ❌ `"재생에너지 관련 기사"`
   - ✅ `"2035년 재생에너지 비중 40% 목표를 제시하고, 계통 연계 지연을 최대 병목으로 지적."`
2. **`excerpt`는 원문에서 그대로 복사.** 의역·창작 금지 → 인용 신뢰성과 "미확인 사실화 방지"의 핵심.
3. **공개·승인 자료만.** 비공개·내부 미승인 정보 금지. 애매하면 `note`에 "추정" 표시.
4. **`id`는 직접 만들지 않는다.** 변환 단계에서 자동 부여(중복 방지).

## 3. 자동 수집 방법론 (RSS/URL → 템플릿)

빠르게 모으는 핵심은 **"수집처를 한 번 등록해두고 스크립트로 반복 수확"** 하는 것입니다.

### 3-1. 수집처 조사 우선순위

1. **RSS가 있으면 최우선.** 대부분의 언론사·일부 기관은 RSS를 제공 → 구조화돼 있어 자동화가 쉽고 안정적.
   - 언론사: 섹션별 RSS(환경/에너지/경제)를 찾아 등록.
   - 확인법: 사이트에서 `rss`, `feed` 검색 / URL 뒤에 `/rss`, `/feed` 시도 / 브라우저 확장으로 피드 탐지.
2. **정책·보도자료 페이지(RSS 없음)**: 부처·공기업 보도자료는 RSS가 없는 경우가 많음.
   - 먼저 RSS 유무를 확인하고, 없으면 (a) 해당 기관의 오픈API/데이터포털 제공 여부 확인, (b) 그래도 없으면 **사이트별 전용 파서**가 필요 → 우선은 수동 수집 대상으로 분류.
3. **인터뷰·기고**: 보통 언론사 RSS/검색으로 함께 잡힘. 발화 주체를 `author`에 명시.

> 팁: 키워드 필터를 활용하면 노이즈가 크게 줄어듭니다. 페르소나 우선순위 키워드를 `sources.csv`의 `keywords`에 `|`로 나열하세요. (예: `요금|원가|재무|계통`)

### 3-2. 수집처 등록 — `data/sources.csv`

`data/sources.example.csv`를 복사해 만듭니다. 컬럼:

| 컬럼 | 설명 |
|---|---|
| `persona` | 기후부/산업부/한전/가스공사 |
| `source_type` | 기고/보도자료/언론기사 … (tier 자동 결정) |
| `org` | 발행처 |
| `feed_type` | `rss` 또는 `html` |
| `url` | RSS 피드 URL 또는 페이지 URL |
| `keywords` | 관련 키워드(`|`로 구분) — 이 중 하나라도 포함해야 수집 |
| `active` | `Y`/`N` (임시 비활성화) |
| `note` | 비고 |

### 3-3. 실행

```bash
# 1) 의존성 설치 (각자 1회)
pip install feedparser requests

# 2) 최근 30일 자료 수집 → data/candidates.csv (템플릿 형식)
python scripts/harvest.py --sources data/sources.csv --out data/candidates.csv --days 30

# 3) (선택) Claude 로 summary/excerpt 초안까지 자동 생성
pip install anthropic
export ANTHROPIC_API_KEY=sk-ant-...   # Windows(파워셸): $env:ANTHROPIC_API_KEY="sk-ant-..."
python scripts/harvest.py --summarize
```

- 결과는 모두 `verified=N`(미검수)로 나옵니다.
- `rss` 소스는 자동 수확, `html` 소스는 "수동 확인 필요" 행으로 표시됩니다.
- URL 기준 **중복은 자동 제거**됩니다.

### 3-4. 자동 요약(--summarize)의 원칙

- 스크립트가 원문 페이지를 받아 Claude(`claude-opus-4-8`)로 `summary/excerpt/stance_signal` **초안**을 만듭니다.
- **어디까지나 초안**입니다. `excerpt`가 원문과 정확히 일치하는지 **사람이 반드시 대조**하세요. (자동 요약도 오류·왜곡 가능)

## 4. 검수 → 승인 → 반영 워크플로

1. `candidates.csv`를 공유 시트에 붙여넣고, 페르소나 오너/검수자가 확인:
   - 출처 실재·링크 접속 OK?  · `summary`가 원문 입장에 충실?  · `excerpt` 원문 일치?  · `tier`·`priority_axis` 적절?  · 중복 아님?
2. 통과분만 `verified=Y`로 바꾸고 **승인 파일/시트**로 이동.
3. 메인테이너가 승인분을 `knowledgeBase.json`으로 변환·커밋. (변환 스크립트는 다음 단계에서 추가 예정 — `scripts/build_kb.py`)
4. **최신성 유지**: 초기 집중 수집 후 **주 1회 harvest 재실행**으로 톱업. 이후 GitHub Actions cron으로 자동화.

## 5. 체크리스트 (수집자용)

- [ ] 담당 페르소나의 커버리지 매트릭스 모든 칸에 ≥1건
- [ ] 모든 행에 필수 컬럼(persona/source_type/tier/title/org/date/url/summary/excerpt/priority_axis/collector/verified) 채움
- [ ] `date`는 `YYYY-MM-DD`
- [ ] `summary`는 입장 중심 2~3문장 (숫자·날짜 포함)
- [ ] `excerpt`는 원문 그대로
- [ ] 공개 자료만, 비공개·미승인 정보 없음
- [ ] URL 중복 없음
