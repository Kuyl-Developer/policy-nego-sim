# E&S 커뮤니케이션 전략 모의 협상 시뮬레이터

대외 발표 **전에**, 커뮤니케이션 전략 초안을 4개 이해관계자 페르소나에게 미리 던져
**반응·수용도·리스크**를 사전 점검하는 웹 도구입니다. (해커톤 초안)

> E&S = Environment & Social. 정부/대외협력 실무자가 실제 발표 전 리스크를 선제적으로 확인하는 것을 목표로 합니다.

## 화면 흐름

1. **페르소나 선택** — 반응을 볼 이해관계자를 고르고, 전략 초안(Markdown)을 입력
2. **다자 채팅형 모의 협상** — 페르소나별 `입장 요약 / 우려·리스크 / 근거(출처 인용)` 표시
3. **평가·개선 리포트** — 종합 수용도 스코어카드 + 이해관계자별 수용도 바 + 개선 전략

## 4개 페르소나

| 페르소나 | 소속 | 강조색 |
|---|---|---|
| 기후부 장관 | 기후에너지환경부 | 초록(green) |
| 산업부 장관 | 산업통상자원부 | 파랑(blue) |
| 한전 사장 | 한국전력공사 | 노랑(yellow) |
| 가스공사 사장 | 한국가스공사 | 하늘(sky blue) |

강조색은 디자인 가이드에 따라 **아이콘/말풍선 테두리에만** 적용됩니다. 폰트는 Pretendard.

## 실행 방법 (빌드 불필요)

정적 웹앱이라 별도 빌드가 없습니다. 로컬 서버로 열기만 하면 됩니다.

```bash
python -m http.server 5173
```

그다음 브라우저에서 <http://localhost:5173> 접속. (ES 모듈 때문에 `file://` 직접 열기보다 로컬 서버 권장)

### 두 가지 동작 모드

- **시드(오프라인) 모드** — 기본값. API 키 없이도 손으로 작성한 페르소나 반응으로 전체 흐름이 완결됩니다.
- **라이브 모드** — 상단 바에 Anthropic API 키를 입력하면, 초안 내용을 반영해 `claude-opus-4-8`(adaptive thinking)로 반응을 동적 생성합니다.
  - 브라우저에서 공식 SDK(`@anthropic-ai/sdk`)를 `esm.sh`로 로드해 호출합니다.
  - ⚠️ 브라우저 직접 호출은 데모 편의용입니다. 실서비스에서는 키를 서버로 옮겨 프록시하세요.

## 안전 제약 (반드시 유지)

디자인 문서의 제약을 프롬프트/UX에 반영했습니다.

- 비공개·내부 미승인 정보 포함 금지, 확인 안 된 발언의 사실화 금지
- 근거는 **지식 베이스 출처만** 인용 (제목 칩 + 링크 + 신뢰도 티어)
- 근거가 부족하면 지어내지 않고 **"공개자료 기반 추정·한계"** 로 표시하고 필요한 추가 데이터를 안내

## 지식 베이스는 현재 "샘플" 입니다

`src/data/knowledgeBase.js` 의 출처는 **데모용 시드/샘플**입니다(`sample: true`).
실제 크롤링 데이터로 교체해야 인용의 신뢰도가 확보됩니다. 신뢰도 티어는
`t1 기고·논문 > t2 보도자료 > t3 일반기사` 순입니다.

## 폴더 구조

```
index.html
styles/app.css
src/
  app.js                # 진입점, 헤더/스텝/라우팅
  actions.js            # 화면 전환 + 시뮬레이션 실행
  data/
    personas.js         # 4개 페르소나 시드
    knowledgeBase.js    # 지식 베이스(샘플 출처 + 신뢰도 티어)
    seedReactions.js    # 오프라인 반응 + 초안 키워드 반영 휴리스틱
  lib/
    anthropic.js        # Claude 연동(SDK/esm.sh) + 시드 폴백
    prompts.js          # 시스템/유저 프롬프트 + 반응 스키마
    store.js            # 상태 저장소
    dom.js              # 초소형 DOM 헬퍼
  components/common.js  # 아바타/입장 배지/출처 칩
  screens/
    selectScreen.js     # ① 선택 + 초안 입력
    chatScreen.js       # ② 채팅형 반응
    reportScreen.js     # ③ 평가·리포트
```
